a11y tests

Thalys modal test

Test description
The homepage will firstly load a cookies message whcih blocks the whole page and all interaction. Then the page shows a 2nd dialog to tell users about the rebrand/merger. Finally, there is a live chat button beneath the 2nd dialog.
Test results

On desktop

The Thalys modal blocks the live chat button and other content. A keyboard may have to TAB many times to reach this modal. A screen-reader user may not know it is there.

However, if the modal is positioned in the DOM immediately after the Cookies modal then then once the user closes the Cookies modal, their very next TAB will take them into the modal.

On Mobile (or zoomed-in desktop)

The Thalys modal covers the screen and a keyboard user may not be able to easily TAB to it to close it and could get lost TABbing to links/form controls that visually sit underneath the modal. A screen-reader user (with vision) may have content read to them beneath the modal but not be able to see it.
