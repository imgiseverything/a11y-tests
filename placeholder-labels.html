a11y tests

Placeholder as labels test

If UX use placeholders instead of labels then we still need accessible names for our inputs.

The 1st field uses a visible label, the 2nd field has a visually hidden label and the 3rd field uses aria-label.

Success criteria

The screen-reader must inform the user what the field is.

Results

VoiceOver (MacOS) reads out the label and the placeholder when the field is empty for options 1 and 2.

JAWS, NVDA, and ChromeVox seem to work as expected in all three cases.

Option 3 (no label and an aria-label instead) is probably best solution as it's the least annoying in Safari (MacOS).
